Grading Philosophy
Grade the QB's action — not the result. A perfect throw that is dropped is still +2. A reckless throw that falls incomplete is still –2. Zero = expected. Most plays should be 0.

The –2 to +2 Scale
+2  Elite
+1  Good / Above Expected
 0  Routine / Expected
–1  Poor / Below Expected
–2  Negative / Game-Harmful
—  Context (no score)

Zero is not average — zero is correct. A QB who executes the play exactly as designed earns a 0 on every column. Positive grades require doing something above what was required. Negative grades mean something below what the play demanded. Reserve +2 and –2 for genuinely exceptional or harmful plays — most plays are 0.

The scale matches the principle used by PFF at the professional level: grade the process and execution, never the outcome. A dropped pass on a +2 throw is still +2. A completion on a –2 decision is still –2.

SOCKS Stage → Column Map

A QB who advances through SOCKS stages should show measurable improvement in specific grade columns. Use this to identify which stage gap a low grade represents, then filter the Drill Library by that stage and category.

Column
SOCKS Element
What a Low Grade Means → Stage to Fix
Stage
Pass Decision
C + K · Count + Key
QB is not reading the key defender correctly or is late triggering. The decision is reactive, not anticipatory.
● Stage 1–2
Run Decision
C + K · Trigger read
Give/keep decision is wrong or made after the mesh point instead of before it. Key read was not resolved pre-snap.
● Stage 1
Pass Release
Timing · Throw Clock
QB is holding the ball past the throw clock window. Pre-snap processing is not complete before the snap — holding is a symptom of a slow scan.
● Stage 2
Pass Pocket
Timing · Pressure Override
QB is bailing or drifting instead of stepping up. Early pressure escape often traces to a slow pre-snap process forcing him to compensate post-snap.
● Stage 2–4
Pass Throw
O + S · Corners + Space
QB is making the right decision but not throwing accurately. Mechanics or release point issue. May also indicate he's throwing to the wrong window despite the correct read.
● Stage 3
Run Fake
Mesh Mechanics
QB is stopping after the handoff. The fake is a block — if nobody bites, it costs the offense a second-level blocker on every run play. Film is the only honest grade.
● Stage 1–3
Pre-Snap Read
S + O + C + K + S · Full scan
QB is missing or skipping the pre-snap scan. A –1 means a specific part of the SOCKS scan failed. A –2 means no visible scan at all. Traces directly to which stage letter is missing.
● All Stages
Ball Placement
S · Space · "Where to throw"
QB read the leverage correctly (good Pre-Snap Read) but delivered to the wrong spot. This is Stage 3 execution failure — leverage tells you where, but the throw didn't go there.
● Stage 3
Pass Decision (blitz plays)
Pressure Override
Pass Decision drops significantly on pressured plays = pressure override not internalized. Hot route is reactive (post-snap) instead of pre-loaded (pre-snap). Identified in Stage 2; mastered in Stage 4.
● Stage 2–4
Adding Pre-Snap Read & Ball Placement to the Google Sheet

The two new columns need to be added to both the Need Graded and Graded Plays tabs. Here's exactly how to do it in about 5 minutes.

1
Open the Need Graded tab
Go to your live grading sheet and click the Need Graded tab. Find column AG (QB NOTE). You're going to insert two new columns before QB NOTE — so QB NOTE slides to column AI and the new fields are AH (Pre-Snap Read) and AI… actually the cleanest approach is to add them after QB IMPACT (col AF) and before QB NOTE. That keeps all grade columns together.
2
Insert two columns before QB NOTE
Click the column letter AG (QB NOTE header) to select the whole column. Right-click → Insert 2 columns left. Two blank columns will appear between QB IMPACT (AF) and QB NOTE. QB NOTE is now column AI.
3
Name column AG — PRE-SNAP READ
Click cell AG1 and type: PRE-SNAP READ
Format it to match your existing headers (same background color as QB IMPACT — light blue). Then add a data validation dropdown to AG2 and below:
Dropdown options (in this order): Elite Read Good Read Routine Read Poor Read Missed / Wrong Read
In Apps Script, the GRADE_MAP should convert these to: +2 / +1 / 0 / –1 / –2. Grade every play — pass and run.
4
Name column AH — BALL PLACEMENT
Click cell AH1 and type: BALL PLACEMENT
Same header formatting. Add dropdown to AH2 and below:
Dropdown options (in this order): Elite Placement Good Placement Routine Placement Poor Placement Negative Placement
Converts to: +2 / +1 / 0 / –1 / –2. Grade completions only — leave blank on incompletions, sacks, and run plays.
5
Mirror the same columns in Graded Plays
Click the Graded Plays tab and repeat steps 2–4 exactly. The column structure must match Need Graded exactly — the Apps Script moveGradedPlays() function copies by column index. If the columns don't match, grades will land in the wrong places.
6
Update the GRADE_MAP in Code.gs (Apps Script)
Open Extensions → Apps Script. Find the GRADE_MAP object and add entries for the new labels so they convert correctly when plays are moved:
// Add inside GRADE_MAP: 'Elite Read': 2, 'Good Read': 1, 'Routine Read': 0, 'Poor Read': -1, 'Missed / Wrong Read': -2, 'Elite Placement': 2, 'Good Placement': 1, 'Routine Placement': 0, 'Poor Placement': -1, 'Negative Placement': -2,
Also update the column index map (the P object at the top of the script) to add: PRE_SNAP_READ: 32, BALL_PLACEMENT: 33 — adjusting the exact numbers to match where the new columns landed (AG = index 32 if columns start at 0).
7
Add to Dashboard (optional but recommended)
In the Dashboard tab, add two new summary rows:
• Pre-Snap Read Avg — average of all PRE-SNAP READ values (all plays)
• Ball Placement Avg — average of all BALL PLACEMENT values (completions only — filter where BALL PLACEMENT is not blank)

These become your SOCKS dashboard metrics — Pre-Snap Read avg should climb as QBs advance stages. Ball Placement avg directly measures Stage 3 leverage-to-throw execution.
